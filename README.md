# Employee Interview Analyzer v2.12.0 — 快速分析模式

## 📌 项目概述

**项目名称：** employee-interview-analyzer（员工访谈结果智能分析技能）  
**当前版本：** v2.12.0  
**版本路径：** v1.0.0 → v2.0.0 → v2.0.1 → v2.1.0 → v2.2.0 → v2.3.0 → v2.4.0 → v2.5.0 → v2.6.0 → v2.7.0 → v2.8.0 → v2.9.0 → v2.10.0 → v2.11.0 → v2.12.0  
**作者：** tommyyang@tencent  
**最后更新：** 2026-03-25  
**配套技能：** [employee-interview-generator v3.1.0](https://github.com/openclaw/workspace/tree/main/skills/employee-interview-generator)

### 核心定位

专为互联网公司 HR 设计的**访谈结果智能分析**技能。输入访谈记录/纪要，自动识别访谈场景，智能匹配分析方法论（≤3种），输出结构化洞察报告。

### 核心能力概览

| 能力 | 说明 |
|------|------|
| 🧠 **智能场景识别** | 关键词+语义+上下文三层识别，支持10种场景 |
| 📐 **7大方法论框架** | 杨三角、韦斯伯德六盒、推拉力、冰山、BEM、职业锚+GROW、STAR |
| ⭐ **方法论精简限制** | 每次分析≤3种（优先2种），深度优先于广度 |
| 📝 **文本自动修正** | 修正错别字、语句不通，提升分析准确性 |
| 👤 **三轨画像系统** | 离职→跳过；招聘→候选人画像；其他→管理干部/员工画像 |
| 🏢 **管理干部画像** | 自动识别管理者身份，生成多场景管理表现画像（对齐HR系统） |
| 📊 **数据可视化** | 雷达图、柱状图、折线图，基于Chart.js |
| 📝 **结论文章** | 800-1500字科学分析文章 |
| 📄 **腾讯文档集成** | 一键生成腾讯在线文档 |
| 🤖 **AI增强维度** | 5大AI时代分析维度与方法论交叉 |

## 🔄 13步分析流程

```
访谈记录输入
    │
    ▼
Step 1: 智能场景识别（自动 + 管理干部身份检测）
    ▼
Step 2: 文本修正（错别字 + 语句不通）
    ▼
Step 3: 方法论自动匹配（优先2种，最多3种）
    ▼
Step 4: 展示分析框架（请用户确认或调整）
    ▼
Step 5: 多维度分析（假设驱动 + 框架分析）
    ▼
Step 6: 交叉验证（矛盾检测 + 一致性）
    ▼
Step 7: 洞察生成（结论 + 建议 + 行动）
    ▼
Step 8: 画像生成 ⭐⭐⭐（非离职场景强制执行）
    │  ├─ 离职访谈 → 跳过
    │  ├─ 招聘面试 → 候选人综合画像
    │  ├─ 管理干部 → 管理干部画像（对齐HR系统）
    │  └─ 非管理者 → 员工画像 + 战场画像
    ▼
Step 9: 组织画像生成（仅组织诊断场景）
    ▼
Step 10/11/13: 批量询问可选步骤（可视化/结论文章/腾讯文档）
    ▼
Step 12: 输出完整报告
```

## 📋 支持的分析场景（10种）

| 分析类型 | 触发场景 | 输入要求 |
|---------|---------|---------|
| **入职访谈分析** | 分析新员工融入情况 | 入职访谈记录 |
| **绩效访谈分析** | 分析绩效问题根因 | 绩效面谈记录 |
| **离职访谈分析** | 分析离职原因 | 离职访谈记录 |
| **晋升访谈分析** | 评估晋升准备度 | 晋升评估面谈记录 |
| **职业发展分析** | 分析职业发展路径 | 职业发展面谈记录 |
| **团队反馈分析** | 诊断团队问题 | 团队访谈/反馈记录 |
| **转岗访谈分析** | 评估转岗适配度 | 转岗面谈记录 |
| **组织诊断分析** ⭐ | 系统性组织健康诊断 | 组织诊断访谈记录（支持多人） |
| **招聘面试分析** ⭐ | 评估候选人匹配度 | 面试记录/评估表 |
| **综合分析** | 多次访谈的趋势分析 | 多份访谈记录 |

> ⚠️ **v2.9.0 移除**：「挽留访谈分析」已被移除，离职分析中可覆盖挽留维度。

## 📐 七大方法论分析框架

| 编号 | 方法论 | 类别 | 核心分析能力 |
|-----|--------|------|-------------|
| M1 | **杨三角理论** ⭐ | 组织诊断 | 从员工思维/能力/治理三支柱诊断组织能力 |
| M2 | **韦斯伯德六盒模型** | 组织诊断 | 从6维度系统评估组织健康度 |
| M3 | **推拉力理论** | 离职分析 | 从推力/拉力/留任力/移动障碍分析离职决策 |
| M4 | **冰山模型** | 人才评估 | 从知识/技能/角色/认知/特质/动机6层评估 |
| M5 | **吉尔伯特 BEM 模型** | 绩效分析 | 从环境(75%)/个人(25%)因素分析绩效差距 |
| M6 | **职业锚理论 + GROW 模型** | 职业发展 | 识别职业锚 + Goal/Reality/Options/Will分析 |
| M7 | **STAR 面试法** | 面试评估 | Situation/Task/Action/Result行为评估 |

## 📋 场景-方法论映射速查表（优先2种，最多3种方法论）

| 场景 | 主方法论(1个) | 辅助方法论(优先1个，最多2个) | 分析重点 |
|------|---------|-----------|---------|
| 入职 | BEM模型(M5) | 冰山模型(M4) | 融入障碍根因 |
| 绩效 | BEM模型(M5) | 杨三角(M1) | 绩效差距根因 |
| 离职 | 推拉力(M3) | 冰山模型(M4) | 离职原因+深层动机 |
| 晋升 | 冰山模型(M4) | BEM模型(M5) | 能力差距 |
| 职业发展 | 职业锚+GROW(M6) | 冰山模型(M4) | 发展方向 |
| 团队反馈 | 杨三角(M1)⭐ | 冰山模型(M4) | 团队问题 |
| 转岗 | 冰山模型(M4) | 职业锚+GROW(M6) | 岗位匹配 |
| 组织诊断 | 杨三角(M1)⭐ | 冰山模型(M4) | 组织健康 |
| 招聘面试 | STAR(M7) | 冰山模型(M4) | 候选人匹配 |
| 综合分析 | 按内容动态匹配 | 最多2个辅助(≤3总计) | 趋势洞察 |

> ⭐ **方法论数量规则**：默认1主+1辅=2种；仅复杂场景扩展至最多3种。  
> ⭐ **组织诊断限制**：杨三角(M1)和韦斯伯德(M2)最多只选一个，优先杨三角。

## 👤 画像系统说明

### 三轨画像系统（v2.8.0+）

| 访谈场景 | 画像策略 | 说明 |
|---------|---------|------|
| **离职访谈** | ❌ 跳过画像 | 聚焦离职原因分析 |
| **招聘面试** | 候选人综合画像 | 含岗位匹配分析+STAR场景呈现 |
| **其他场景** | 管理干部 or 员工画像 | 基于Step 1管理干部身份检测自动切换 |

### 管理干部画像（v2.11.0 对齐HR系统）

管理干部画像包含两大部分：

1. **基础画像**（7维度）：工作投入度、异动风险、战场、战场匹配度、风险与挑战、个人优劣势、个人发展诉求
2. **多场景管理表现画像**（6大场景）：
   - 业务管理 / 团队管理 / 周边协作 / 资源使用 / 自我管理 / 其他
   - 行为标签：`正向行为` / `需关注和待提升行为` / `无标签`（对齐HR系统三分类）
   - 场景类型可多选，每条行为独立编号
   - 行为描述严格按 STAR 原则（Situation→Task→Action→Result）
   - 新增「发生时间」字段

### 候选人综合画像（v2.8.0+，招聘面试专用）

含基础信息、能力优劣势、发展潜力评估、个人发展诉求、岗位匹配分析矩阵、STAR实际工作场景呈现、综合评价及录用建议。

### 员工画像

含基础信息、能力特征、性格与行为模式、战场画像、个人优劣势、个人发展诉求、发展潜力与风险。

## 🚀 使用方式

### 最简用法
```
"帮我分析这份访谈记录：[粘贴访谈内容]"
```
→ 自动执行13步分析流程，Step 9 完成后一次性询问可选步骤

### 指定场景
```
"这是一份绩效访谈记录，帮我分析绩效下滑的根因：[粘贴内容]"
```

### 指定方法论（最多3种）
```
"用推拉力理论分析这份离职访谈：[粘贴内容]"
```

### 多方法论交叉（≤3种）
```
"同时用杨三角和韦斯伯德分析这份组织诊断：[粘贴内容]"
```

### 多文件综合
```
"这5份组织诊断访谈，帮我做综合分析：[上传/粘贴多份记录]"
```

### 腾讯文档链接输入（v2.9.0+）
```
"帮我分析这份访谈记录：https://docs.qq.com/doc/xxx"
```
→ 自动通过 tencent-docs skill 读取文档内容后分析
